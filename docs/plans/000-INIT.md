
# Triggity

Description: Triangular affect journaling

Inspired by: actualfreedom.com.au categorization of feelings into two categories: 'good' feelings; 'bad' feelings; felicitous/ innocuous/ naive feelings.

UX: 
- Present a triangular "surface" to the user who then taps a point on the inside depending on how they feel right now. No words, labelling. Just the location of this point is logged; and this location indicates where in the good/bad/naive specturm they are affective. 
- Optional text box to log any thing (event, etc.) at the same timestamp as affect logging. Affect logging is mandatoruy; text is just secondary, optional.
- Trends over time/ stats

Tech: 
- PWA TypeScript + SolidJS + TailWind + flake.nix (exactly like github.com/juspay/kolu).
- SQLite + "Local first" that syncs with sqlite in browser???
